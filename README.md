# 乡镇小店智能经营平台

面向乡镇小型超市/便利店的轻量化数字经营平台。项目从真实线下门店场景出发，先建设店主经营后台，解决商品价格难记、库存不清、销售数据缺失、临期商品难管理等问题；后续逐步开放顾客小程序端，并接入大模型能力，提供经营分析、补货建议、临期处理建议和营销文案生成。

## 项目背景

乡镇小超市通常存在以下问题：

- 商品 SKU 多，店主和家人很难记住全部价格、进价、库存和货架位置。
- 销售记录依赖人工记账或收银流水，难以统计真实毛利和畅销/滞销商品。
- 库存缺少实时管理，容易出现缺货、压货和临期损耗。
- 门店线上化能力弱，特价商品、到货信息和顾客预留无法形成闭环。
- 店主缺少经营分析工具，难以基于数据做补货、促销和清仓决策。

本项目希望把一家线下小超市逐步改造成“内部经营后台 + 顾客小程序 + AI 经营助手”的智能零售平台。

## 产品定位

**一句话定位：**

为乡镇小店提供商品查价、库存管理、销售记录、临期预警、经营看板和 AI 决策辅助能力，并逐步支持线上特价展示与到店自提。

**目标用户：**

- 小超市/便利店店主
- 家庭经营成员
- 未来的小程序顾客

## MVP 范围

第一阶段只做店主后台，不做完整商城，不做复杂会员系统。

MVP 目标：

> 店主可以录入商品，搜索或扫码查价，记录销售，系统自动扣减库存，并提醒低库存和临期商品。

核心功能：

- 商品管理：新增、编辑、删除、搜索商品。
- 快速查价：按商品名、条码、分类、品牌查找价格和库存。
- 销售记录：创建销售单，记录商品、数量、售价、毛利。
- 库存扣减：销售后自动减少库存，并写入库存流水。
- 低库存提醒：库存低于阈值时进入预警列表。
- 临期提醒：根据到期日期提示即将过期商品。
- 经营看板：展示销售额、毛利、库存成本、畅销商品等基础指标。

## 推荐开发顺序

1. 编写产品说明、业务流程和数据模型。
2. 搭建 Next.js + TypeScript + Tailwind + Prisma + PostgreSQL 项目。
3. 实现商品管理模块。
4. 实现快速查价模块。
5. 实现销售记录与库存扣减。
6. 实现低库存和临期提醒。
7. 实现经营看板。
8. 实现特价活动管理。
9. 开放顾客端商品和特价展示。
10. 接入 AI 经营助手。

## 项目亮点

- 基于真实线下小超市场景，而不是普通商城 Demo。
- 覆盖商品、库存、销售、临期、活动、顾客预留和 AI 分析等完整业务链路。
- 使用库存流水追踪库存变化，便于审计和经营复盘。
- 预留小程序端和 AI 能力，具备从内部后台演进为线上平台的扩展路径。
- PostgreSQL 可作为经营数据分析底座，后期可结合 pgvector 扩展 AI 记忆与检索能力。

## 本地开发

### 前置条件

- Node.js ≥ 22
- pnpm ≥ 9
- Docker（用于本地 PostgreSQL）

### 启动步骤

```bash
# 1. 安装依赖
pnpm install

# 2. 启动 PostgreSQL
docker compose -f docker-compose-environment.yml up -d

# 3. 应用迁移 + 灌演示数据
pnpm prisma migrate deploy
pnpm db:seed

# 4. 启动开发服务器
pnpm dev
# 访问 http://localhost:3000
```

### 常用脚本

```bash
pnpm dev              # 启动 dev server
pnpm build            # 生产构建
pnpm start            # 启动生产 server
pnpm lint             # ESLint
pnpm tsc --noEmit     # 类型检查
pnpm db:seed          # 重灌演示数据（幂等）
pnpm db:reset         # 清库 + 重灌（破坏性）
pnpm smoke            # 跑 curl 烟测（需 dev/start 在跑）
```

### 测试与 CI

- `pnpm smoke` 调用 `scripts/smoke.sh`，覆盖 30 个 API + 页面端到端用例
- 每次 push / PR 触发 `.github/workflows/ci.yml`：build → 类型检查 → lint → migrate → seed → 启动 prod server → 烟测
- 详细测试矩阵见 `docs/testing.md`
